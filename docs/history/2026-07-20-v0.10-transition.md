# v0.10 transition — 2026-07-20

The record of the canonical repository adopting the opt-in **v0.10** authenticated
chain for its own releases (#76). Where the [recurring dogfood
ceremony](2026-07-20-recurring-dogfood.md) proved the wired chain in a throwaway
repo with throwaway keys, this is the real thing: a signed `release/v0.2` genesis,
cut with the maintainer's key on the canonical repo, that **continues the published
line** (`v0.1.0`–`v0.2.1`, default `release/v0.1` chain) to **`v0.3.0`**.

> **This is a real release of `semver-trust-go`.** The `v0.3.0` tag and its
> `release/v0.2` attestation are published upstream and CI-verified. The earlier
> `v0.1.0`–`v0.2.1` tags and their `release/v0.1` attestations are untouched — this
> **continues** the version line, it does not replace or re-cut them.

## How it was cut

A **collaborative ceremony** — the agent prepared, verified, and handed off exact
commands; the maintainer ran the two signing steps and set the CI variable. The agent
signed nothing on the real chain (the agent contract forbids `attest`/`release`; signing
is a human accountability act, specification Principle 2).

1. **Review.** `main` was 117 commits past `v0.2.1`, entirely unreviewed; five touched
   meta-paths (`.github/workflows/**`) and **hard-aborted verification (§5.4)**. One
   maintainer-signed review over `v0.2.1..main` (reviewer `brad.pinter@gmail.com`,
   approved) lifted the agent-authored body to **T2** (agent authorship + distinct human
   review, ADR-025) and cleared the meta-path gate — one attestation covering the range.
2. **Descriptor.** An out-of-band bootstrap descriptor (ADR-028), authored by the
   deterministic digest recipe in the [runbook](../recurring-release-runbook.md), supplied
   from **outside** the repository. `interval_mode: inception`, `version_predecessor` set
   to the `v0.2.1` binding (`ref_oid 47c075dc…`, `commit_oid 66323e6e…`), and
   `trust_material` for **all three** declared registries (`allowed_signers`,
   `gpg-keyring.asc`, `attestation_signers`).
3. **Genesis release.** `release --predicate v0.2 --bootstrap-descriptor <desc>
   --claimed-bump minor --blast low --verify-time 2026-07-21T00:00:00Z`, signed with the
   maintainer's attestation key (SSH `SHA256:1NVcUFjQ+j7TKWPluo59uht1h3IDypfTg17dGX2420A`).
   The `release/v0.2` attestation is stored under both the commit and the `v0.3.0` tag.
4. **Publish.** Attestation refs first, then the tag — the tag push triggered
   `release.yml`.

## CI verified it

`release.yml` run
[`29791654844`](https://github.com/semver-trust/semver-trust-go/actions/runs/29791654844)
(three jobs green). The **verify (enforced dogfood gate)** job materialized the
`BOOTSTRAP_DESCRIPTOR` repository variable, ran `verify --chain-head` — which
self-authenticates the descriptor against the tag's tree, binds the genesis to the
descriptor's identity, asserts the accepted chain head **is** the pushed tag, and reads
the badge trust from the *verified* head — and logged `Verified at effective T2.` The
badge and goreleaser jobs then published.

## Reproduced by an outsider

A fresh public clone + fetched attestation refs + the out-of-band descriptor reproduces
the head independently at the recorded instant:

```sh
git clone https://github.com/semver-trust/semver-trust-go /tmp/fresh
cd /tmp/fresh
git fetch origin 'refs/attestations/*:refs/attestations/*'
go run ./cmd/semver-trust verify --repo . --to v0.3.0 \
  --bootstrap-descriptor <descriptor-outside-the-clone> \
  --verify-time 2026-07-21T00:00:00Z --chain-head
# accepted chain head: v0.3.0 -> 7d23a679… (effective T2, §7.5/ADR-029)
```

## Operational note — the descriptor must survive the CI variable round-trip

`release.yml` materializes the descriptor with `printf '%s'` (no added newline), and the
genesis binds its authority identity to the **exact** descriptor bytes. GitHub Actions
strips a trailing newline from a stored variable value, so a descriptor file ending in
`\n` yields a variable whose digest differs from the genesis's — and `verify --chain-head`
fails in CI. The fix is to bind the genesis to the **canonical, no-trailing-newline**
descriptor — byte-identical to what the variable stores — and verify the round-trip
(`sha256` of the read-back variable value == the genesis's bound identity) **before**
pushing the tag. That equivalence check is the pre-publish gate.
